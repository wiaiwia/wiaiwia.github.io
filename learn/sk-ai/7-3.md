### 7월 16일

#### AI literacy

1. 생성형 AI 는 컨텍스트 윈도우가 있다.
- 작업 기억의 한계 때문에 가장 앞 부분과 가장 끝 부분의 내용이 컨텍스트에 많이 담긴다.
- 많아지면 핵심을 요약해 새 대화로 옮기는 방향이 좋다.

2. 기존의 AI LLM을 넘어선 AI AGENT.
- 목표가 주어지면 스스로 계획, 도구 사용, 실행을 하고 스스로 검증한다.
- Codex, Claude code
- Single agent와, Multi agent가 있다.
- Multi agent는 Orchestrator가 agent에게 역할 분담을 시킨다. 모두가 공유하는 지속메모리가 있고, 상황을 보며 일을 쪼개고 다시 나눈다.
- 그 역할의 크기, 얼마나 스스로 하냐를 agency 라고 한다.

3. video ai에 물리 엔진을 입혀 실제 세계를 만드는 것을 World Model이라고 하고 google의 genie와 nvidia의 cosmos가 있다.

4. notebookLM : 내가 올린 자료에서만 근거에서 답함(rag)
perplexity: 검색을 통한 답, 출처를 달아줌
liner scholar: 학술 논문만 검색한 답을 보여줄 수 있음

5. 프롬프트에게 어떤 패턴을 보여주며 예시를 드는 것을 few-shot이라고 한다.

#### Codex

1. 터미널에서 돌릴 수 있음.

2. 프로젝트 루트에 agents.md 파일에 지침을 적을 수 있음. 웹 기반에서 시스템 프롬프트 같은 느낌.
실제 프로젝트에서는 Andrej Karpathy의 작업 원칙을 넣음
(Think before coding, Simplicity first, Surgical changes, Goal-driven execution)
Surgical changes = 요청과 직접 관련된 코드만 건드리는 것. 요청에 의해서만 코드를 바꾼다.

3. MCP(Model Context Protocol)
-ai agent가 외부 도구나 데이터에 연결하는 표준 protocol이고 codex는 하나의 mcp client라고 볼 수 있음.

4. Skill
-검증된 작업 절차, 노하우를 에이전트에게 가르치는 재사용 지침 묶음.
-상황에 따른 workflow를 찾아 가게 함
-MCP가 도구라면, Skill은 방법, 절차.

5. Playwright MCP
-에이전트가 브라우저를 직접 조작할 수 있게 함.
-이동, 클릭, 입력, 폼 제출, 스냅샷 등
-화면 이미지가 아닌 코드 베이스의 접근.

6. Vercel
-만든 앱(웹)을 온라인으로 배포. MCP의 일종이고 원격(HTTP) + OAuth 서버

7. Context7 MCP
-각종 라이브러리 프레임워크(Next.js, supabase같은) 최신버전별 공식 문서 예제 코드 가져와 에이전트 컨텍스트에 삽입. 오래된 데이터로 인한 환각 방지

8. Superpowers
-스킬 프레임워크
-브레인 스토밍, 체계적 디버깅, Test driven development, 계획 작성 같은 검증된 workflow를 Skill로 묶음. 작업에 맞게 자동으로 에이전트를 이끌음
